# Project Proposal: Smart Inventory Management System

**Submitted by:** Development Team, IT Department
**Date:** April 20, 2026
**Version:** 1.0

---

## Background

The company's current inventory management system is based on Excel spreadsheets and a legacy ERP system implemented in 2018. In recent years, with 150% growth in business volume, the system has become a significant bottleneck. Inventory errors cause annual losses of approximately NIS 800,000, and order processing time averages 48 hours instead of the required 12 hours.

## Project Objectives

The project aims to replace the existing inventory management system with a cloud-based smart system that provides real-time tracking, AI-powered demand forecasting, and full integration with logistics and sales systems. The system is expected to reduce inventory errors by 90%, shorten order processing time to 4 hours, and save the company approximately NIS 600,000 per year.

## Project Scope

### Phase 1 — Infrastructure and Migration (Months 1–3)

The first phase will focus on establishing the technology infrastructure on AWS, migrating historical inventory data, and building API interfaces for integration with the existing ERP system. At the end of this phase, all data will be available in the new system in parallel with the legacy system.

### Phase 2 — Core Development (Months 3–6)

This phase will build the core modules: real-time inventory management with barcode scanning, automated alerts for critical stock levels, a management dashboard with built-in BI, and an automated supplier ordering interface. Additionally, an AI model for demand forecasting will be developed based on historical sales data, seasonality, and market trends.

### Phase 3 — Testing and Deployment (Months 6–8)

The final phase will include comprehensive testing (unit, integration, and load), user training, parallel operation of both systems for one month, and full transition to the new system.

## Project Team

The project requires a team of 4 developers (2 Backend, 1 Frontend, 1 DevOps), a project manager, an AI/ML specialist (external consultant), and a systems analyst. The proposed project manager is Tal Oren from the IT department, with 8 years of experience managing software projects.

## Risks

The primary risk is data migration delays due to low data quality in the existing system. To mitigate this, a preliminary data cleaning phase is planned for the first two weeks. An additional risk is employee resistance to change, which will be addressed through a comprehensive training program and involving key users in the development process from the earliest stage.

## Expected ROI

With projected annual savings of NIS 600,000 from inventory error reduction, NIS 200,000 from process optimization, and NIS 150,000 from reduced excess storage costs, the investment is expected to pay for itself within 20 months. After that, net annual savings are projected at approximately NIS 950,000.
